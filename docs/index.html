zui

nikitazu

Zuev’s User Interface library.

Please refer to Syntax module documentation in order to see how to start with creating declarative user interfaces.

After that read the Notify Changes module to find out how to create a view model with data binding.

Let us review a simple example of defining a frame with declarative UI.

(require racket/gui
         zui/syntax)
 
(define frame
  (zui frame% ([label "Test"]
               [width 320]
               [height 240])
       (zui vertical-pane% ([alignment '(center center)])
            (zui message% ([label "Hello, declarative User Interface!"]))
            (zui message% ([label "I love you so much!"]))
            (zui button% ([label "Quit"]
                          [callback (λ (s e) (exit))])))))
 
(send frame show #t)

First expression in example is:

(require racket/gui
         zui/syntax)

Here we require racket/gui module which provides standard racket GUI primitives and zui/syntax module which provides zui macro.

In the next expression we define an instance of a frame% with declarative user interface.

(define frame
  (zui frame% ([label "Test"]
               [width 320]
               [height 240])
       (zui vertical-pane% ([alignment '(center center)])
            (zui message% ([label "Hello, declarative User Interface!"]))
            (zui message% ([label "I love you so much!"]))
            (zui button% ([label "Quit"]
                          [callback (λ (s e) (exit))])))))

Let us review a single macro expression which creates a message% object.

(zui message% ([label "Hello, declarative User Interface!"]))

First argument is class identifier: message%.

Then goes a list of init arguments: ([label "Hello ..."]).

After a list of init arguments we can put any number of sub-expressions within the zui macro.

(zui vertical-pane% ([alignment '(center center)])
     (zui message% ([label "Hello..."]))
     (zui message% ([label "I love..."]))
     ...)

As in the example above the zui macro would handle the parent/child relationships between the classes automatically.

The last expression shows the frame% on screen.

(send frame show #t)
